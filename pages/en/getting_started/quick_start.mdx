## Step 1: Understand the Pipeline Configuration Structure

Below is a minimal example configuration file: examples/sayhello.yaml

```yaml
servers:
  sayhello: servers/sayhello

pipeline:
  - sayhello.greet
```

This configuration file contains two main parts:

| Field    | Description                                                                 |
| -------- | --------------------------------------------------------------------------- |
| servers  | Declares the Server modules used in this workflow. This example uses a toy server named sayhello. |
| pipeline | Defines the execution flow. Each item indicates a Tool registered in a Server to be called. Here, the greet tool is called. |

## Step 2: Implement an MCP Server

sayhello is a minimal MCP Server with a single Tool named `greet`:

```python
from typing import Dict
from ultrarag_mcp.server import UltraRAG_MCP_Server
from tabulate import tabulate

app = UltraRAG_MCP_Server("sayhello")

@app.tool(output="name->msg")
def greet(name: str) -> Dict[str, str]:
    ret = f"Hello, {name}!"
    resp_tab = tabulate(
        [
            ["Server", __file__],
            ["Input", name],
            ["Output", ret],
        ],
        headers="firstrow",
        tablefmt="fancy_grid",
    )
    app.logger.info("\n" + resp_tab)
    return {"msg": ret}

if __name__ == "__main__":
    # Start the sayhello server using stdio transport
    app.run(transport="stdio")
```
You can see:

- The tool `greet` takes a `name` as input
- The returned greeting is named `msg`
- The decorator `@app.tool(output="name->msg")` defines the input and output fields

## Step 3: Build the Parameter File

Before running the code, it is natural to configure parameters. UltraRAG provides a convenient way to configure parameters by consolidating all you need into a single YAML file. Simply run the following command:

```shell
ultrarag build examples/sayhello.yaml
```

Terminal output:

![](/images/getting_started/sayhello_build.png)

You will get examples/parameters/sayhello_parameter.yaml with the following content:

```yaml
sayhello:
  name: UltraRAG 2.0
```

Fill in the user name you want to pass to greet here.

## Step 4: Run the Pipeline

Once everything is ready, run the following command to execute the full workflow:

```shell
ultrarag run examples/sayhello.yaml
```

You will see output similar to this in the terminal and log file:
