## 数据格式说明

我们建议用户将所有测试数据处理为 .jsonl 格式，并遵循以下结构规范，以保证与 UltraRAG 的各模块兼容：

非选择题数据格式：

```json
{
  "id": 0,  // 整数编号
  "question": "xxxx",  // 问题文本
  "golden_answers": ["xxx", "xxx"],  // 标准答案列表，可包含多个
  "metadata": { ... }  // 其他信息字段，可选
}
```

选择题数据格式：

```json
{
  "id": 0,
  "question": "xxxx",
  "golden_answers": ["A"],  // 标准答案为选项字母（如 A–D）
  "choices": ["xxx", "xxx", "xxx", "xxx"],  // 选项文本列表
  "metadata": { ... }
}
```

语料库数据格式：

```
{
  "id": "0",
  "contents": "xxxxx"  // 语料分块后的文本片段
}
```
