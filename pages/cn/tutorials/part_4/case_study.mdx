UltraRAG 2.0 提供了便捷的 Case Study 可视化机制，帮助科研人员快速检查和分析自己构建的 pipeline 是否符合预期需求。

在运行完成 pipeline.yaml 文件后，系统会在 output 文件夹下自动生成一份 memory 日志文件，例如：`output/memory_nq_rag_20250818_122640.json`。只需执行以下命令，即可启动 Case Study 可视化网站：

```shell
python ./script/case_study.py \
  --data output/memory_nq_rag_20250818_122640.json \
  --host 0.0.0.0 \
  --port 8080 \
  --title "Case Study Viewer"
```

运行成功后，你可以通过浏览器访问相应地址，进入 Case Study Viewer 界面。该界面将以直观的可视化方式展示 Pipeline 各阶段的输入输出与 Memory 追踪信息，方便逐步检查推理链条与中间状态。界面示例如下图所示：

![](/images/part_4/case_study.png)

研究人员可以逐条切换不同的 Case，直观对比 输入问题、检索结果与模型输出 的对应关系，从而更高效地诊断潜在问题，优化 Pipeline 设计，并验证整个推理流程是否符合预期。
